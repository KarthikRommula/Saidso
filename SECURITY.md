# Security policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 0.5.x (latest) | Yes |
| < 0.5 | No |

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email **eigenhmax@gmail.com** with the subject line `saidso security`. Include:

1. A description of the vulnerability and its potential impact.
2. Steps to reproduce or a proof-of-concept.
3. The saidso version and Python version affected.

You will receive an acknowledgement within 48 hours. If the report is confirmed,
a patched release will be published and you will be credited (unless you prefer
to remain anonymous).
